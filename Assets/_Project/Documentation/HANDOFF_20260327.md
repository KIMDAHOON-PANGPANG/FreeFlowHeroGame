# FreeFlowHeroGame 인수인계 문서

> 최종 업데이트: 2026-03-27
> 작성 목적: 다른 기기(노트북 등)에서 이어서 작업할 수 있도록 현재 프로젝트 상태를 정리

---

## 1. 프로젝트 개요

- **목표**: 인디게임 REPLACED(Sad Cat Studios)의 Batman Arkham 스타일 프리플로우 전투를 Unity 2D 횡스크롤로 완벽 재현
- **엔진**: Unity 2022.3+ LTS, 2D URP
- **입력**: New Input System (Legacy 사용 금지)
- **프레임**: 60fps 고정 (프레임 기반 전투 판정)
- **애니메이션 에셋**: EEJANAI_Team FreeFighterAnimations (18종 FBX)

## 5. ActionEntry 데이터 구조 (최신)

```csharp
public class ActionEntry
{
    // ─── 기본 정보 ───
    public string id;           // "LightAtk1"
    public string name;         // "기본공격 1타"
    public string clip;         // 애니메이션 클립명

    // ─── 프레임 데이터 (60fps) ★ 데이터 튜닝 ───
    public int startup;         // 선딜
    public int active;          // 히트 판정 활성
    public int recovery;        // 후딜

    // ─── 재생 제어 ★ 데이터 튜닝 ───
    public float playbackRate;  // 1.0 = 정속 (0.6 = 느리게)
    public float transitionIn;  // 이전→이 액션 블렌딩(초) [NEW 2026-03-27]
    public float transitionOut; // 이 액션→다음 블렌딩(초)  [NEW 2026-03-27]

    // ─── 캔슬 시스템 ───
    public float cancelRatio;   // Recovery 캔슬 비율 (0~1)
    public CancelRoute[] cancels; // 입력별 분기
    public string defaultNext;  // 기본 전이

    // ─── 이동/태그 ───
    public float moveSpeed;     // 액션 중 이동 속도
    public string[] tags;       // "light", "heavy", "dodge" 등

    // ─── 타임라인 노티파이 ───
    public ActionNotify[] notifies;
}
```

## 6. 노티파이 타입 일람

| 타입 | 역할 | 비고 |
|------|------|------|
| `STARTUP` | 선딜 구간 표시 | 자동 생성 |
| `COLLISION` | 히트박스 활성 구간 | 필수 |
| `CANCEL_WINDOW` | 캔슬 가능 구간 | cancelRatio로 자동 계산 가능 |
| `PENDING_WINDOW` | 캔슬 전 입력 수집 구간 | COLLISION 종료 ~ CANCEL 시작 사이 |
| `WARP` | 워핑(적에게 순간이동) 타이밍 | 거리 비례 시간 |
| `INVINCIBLE` | 무적 프레임 | Dodge용 |
| `SUPER_ARMOR` | 슈퍼아머 | Heavy Attack용 |
| `PROJECTILE` | 투사체 발사 | Huxley용 |
| `CAMERA_SHAKE` | 카메라 흔들림 | 연출용 |
| `SFX` / `VFX` | 효과음/이펙트 | 연출용 |

## 7. 최근 작업 이력 (2026-03-27 세션)

### 완료된 작업

1. **PENDING_WINDOW 우클릭 메뉴 추가**
   - `ActionTableEditorWindow.cs`의 타임라인 우클릭 메뉴에 PENDING_WINDOW 노티파이 추가 항목 구현

2. **에디터 프리뷰 ↔ 인게임 포즈 불일치 원인 분석**
   - **원인**: Animator 트랜지션 크로스페이드 블렌딩
   - 에디터는 `AnimationMode.SampleAnimationClip`으로 직접 샘플링 (트랜지션 없음)
   - 인게임은 `Animator` 상태머신으로 전환 (크로스페이드 있음)
   - 그래서 같은 wallTime에서 클립 내 포즈가 다르게 보임

3. **트랜지션 블렌딩 보정 기능 구현**
   - `ActionData.cs`에 `transitionIn`/`transitionOut` 필드 추가
   - 인스펙터에 슬라이더 UI 추가 (0.0~0.5초)
   - `SamplePreviewAnimation()`에서 `transitionIn`만큼 샘플 시간 오프셋 적용
   - 수식: `sampleTime = previewFrame / 60f - transitionIn`

4. **디버그 로그 정리**
   - `StrikeState.cs`: `[InGame Anim]` 진단 로그 제거
   - `ActionTableEditorWindow.cs`: `[Editor Preview]` 진단 로그 제거

### 수정된 파일 (이번 세션)

| 파일 | 변경 내용 |
|------|----------|
| `ActionData.cs` | transitionIn/transitionOut 필드 추가 |
| `ActionTableEditorWindow.cs` | 우클릭 PENDING_WINDOW, 블렌딩 인스펙터 UI, 프리뷰 보정, 진단 로그 제거 |
| `StrikeState.cs` | 진단 로그 제거 |

## 8. 현재 미해결 이슈

### 이슈: LightAtk4 (4타) 모션 이상

- **증상**: 4타 니킥 모션이 이상하게 보임 (구체적 증상 미확인)
- **가능한 원인**:
  1. **COLLISION 시작이 늦음** — 프레임 21 (1타:10, 2타:12, 3타:16)
  2. **이동 속도 차이** — 4타만 5.0 (1~3타는 6.0)
  3. **트랜지션 블렌딩** — transitionIn이 0으로 세팅되어 있어 크로스페이드 보정 안됨
  4. **AnimatorController** — 모든 콤보 트랜지션이 동일하게 0.05초 크로스페이드
- **다음 조치**: 구체적으로 어떤 식으로 이상한지 확인 후 대응 필요
