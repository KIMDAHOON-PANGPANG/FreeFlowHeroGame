# 프리플로우 전투 시스템 — 현재 상태 정리

> REPLACED(Sad Cat Studios) 전투 시스템 Unity 2D 횡스크롤 재현 프로젝트

## 완료된 시스템

### Phase 1 — 기초 ✅
- PlayerCombatFSM (상태 머신: Idle, Strike, Guard, Dodge, Execution)
- Hitbox/Hurtbox 시스템 (OverlapBox 기반 — COLLISION 노티파이 직접 판정)
- CombatEventBus (모듈 간 이벤트 통신)
- 기본 Strike (공격 상태)

### Phase 2 — 프리플로우 코어 ✅
- 워핑 (WARP 노티파이 기반, 자동 타겟 + 오프셋 + 이징)
- 콤보 체인 (LightAtk1→2→3→4→1 루프, 인풋 버퍼, 캔슬 윈도우)
- Dodge (회피 + 무적 프레임)
- 가드 + 가드 카운터 (가중 랜덤 모션)

### Phase 3 — 적 AI ✅
- 적 충돌 수정 (Kinematic Rigidbody2D, 수동 충돌)
- AnimatorController 매핑 (EEJANAI 클립 리타겟)
- AttackCoordinator (FIFO 공격 스로틀링, 동시 2명 제한, 호흡 시간)
- 기본 AI 루프: Idle → Patrol → Chase → Telegraph → Attack → PostAttack → Chase

### 히트 판정 시스템
- **플레이어 공격**: COLLISION 노티파이의 hitboxOffset/Size로 Physics2D.OverlapBox 직접 판정
- **적 공격**: 동일 방식 (EnemyAIController.TryCollisionHit)
- 액션별 히트 범위 개별 조절 가능 (Action Table Editor)
- 복수 적 동시 히트 지원

### 타겟팅 시스템
- 첫 공격 (Idle→Strike): 방향 무관, 순수 거리 기준 가까운 적 선택
- 콤보 중 (WARP 리타겟): 현재 facing 유지, 같은 적 계속 공격
- 방향키 입력 시: 해당 방향 적 우선
- 관통 방지: 경로 상 다른 적이 있으면 가까운 적 우선 선택
- 처형/가드카운터: 방향 무관 가까운 적

### 워핑 시스템
- WARP 노티파이 파라미터: offsetX, offsetY, speed, duration, easeType, minRange, maxRange
- 워핑 후 ROOT_MOTION 클램프: 적과 최소 거리(warpMinContactDistance) 유지
- BattleSettings에서 Min Contact Distance 조절 가능

### 액션 테이블 (JSON 기반)
- PC_Hero.json: 플레이어 전체 액션 정의
- Enemy_Grunt.json: 적 액션 정의
- Action Table Editor (REPLACED > Action Table Editor): GUI 편집 도구
- 노티파이 타입: COLLISION, WARP, ROOT_MOTION, STARTUP, CANCEL_WINDOW, PENDING_WINDOW, HOMING
